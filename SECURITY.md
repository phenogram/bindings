# Security policy

[Русская версия](#политика-безопасности)

## Supported versions

Security fixes target the latest release and the default branch.
Older releases can remain unsupported.

## Report a vulnerability

Use a [private GitHub security advisory](https://github.com/phenogram/bindings/security/advisories/new).
Do not open a public issue for an undisclosed vulnerability.

Include these details:

- Affected version or commit
- Impact
- Reproduction steps
- Minimal proof of concept
- Suggested fix, if available

Remove all bot tokens and personal data from the report.
Revoke a bot token immediately after an accidental disclosure.

The maintainer will assess the report.
The maintainer can request more information.
Coordinate public disclosure with the maintainer.

# Политика безопасности

[English version](#security-policy)

## Поддерживаемые версии

Исправления безопасности предназначены для последнего релиза и основной ветки.
Старые релизы могут не поддерживаться.

## Сообщение об уязвимости

Используйте [приватное уведомление GitHub](https://github.com/phenogram/bindings/security/advisories/new).
Не создавайте публичный issue для ещё не раскрытой уязвимости.

Добавьте следующие данные:

- Затронутая версия или коммит
- Последствия
- Шаги воспроизведения
- Минимальный пример
- Предлагаемое исправление при наличии

Удалите из отчёта все токены ботов и персональные данные.
Немедленно отзовите случайно раскрытый токен бота.

Сопровождающий оценит отчёт.
Сопровождающий может запросить дополнительную информацию.
Согласуйте публичное раскрытие с сопровождающим.
