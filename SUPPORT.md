# Support

[Русская версия](#поддержка)

## Get help

Read the [English documentation](docs/en/getting-started.md).
Run the [offline examples](examples/README.en.md).
Search the [existing issues](https://github.com/phenogram/bindings/issues).

Open an issue when you find a reproducible package defect.
Use a minimal PHP example.
Include the package version, PHP version, and Telegram Bot API method.
Remove all tokens and personal data.

Use [Phenogram Framework](https://github.com/phenogram/framework) support for framework routing, middleware, and runtime questions.
Use the [official Telegram Bot API documentation](https://core.telegram.org/bots/api) for Telegram platform behavior.

Report a vulnerability through the private process in [SECURITY.md](SECURITY.md).
Do not disclose a vulnerability in a public support issue.

# Поддержка

[English version](#support)

## Получение помощи

Прочитайте [русскую документацию](docs/ru/getting-started.md).
Запустите [примеры без сети](examples/README.md).
Проверьте [существующие issues](https://github.com/phenogram/bindings/issues).

Создайте issue, если нашли воспроизводимую ошибку пакета.
Добавьте минимальный пример PHP.
Укажите версию пакета, версию PHP и метод Telegram Bot API.
Удалите все токены и персональные данные.

Используйте поддержку [Phenogram Framework](https://github.com/phenogram/framework) для вопросов о маршрутах, middleware и среде выполнения.
Используйте [официальную документацию Telegram Bot API](https://core.telegram.org/bots/api) для вопросов о поведении платформы Telegram.

Сообщайте об уязвимостях через приватный процесс из [SECURITY.md](SECURITY.md).
Не раскрывайте уязвимость в публичном issue поддержки.
