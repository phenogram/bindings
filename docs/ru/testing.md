# Тестирование

Стандартным тестам не нужен токен бота.
Они не отправляют сетевые запросы.

## Установка зависимостей для разработки

Запустите команду из корня пакета:

```bash
composer install
composer tools:install
```

Репозиторий не хранит корневой файл `composer.lock`.
Composer разрешает корневые зависимости для разработки по ограничениям версий.
Отдельные проекты инструментов хранят lock-файлы для PHPStan и PHP CS Fixer.
Эти файлы не изменяют разрешение зависимостей для пользователей пакета.

## Запуск PHPUnit

Запустите все тесты без сети:

```bash
vendor/bin/phpunit
```

Тесты проверяют поведение сериализатора.
Они проверяют свои типы результата.
Они также запускают каждый файл из `examples` в отдельном процессе PHP.

## Запуск примеров

Запустите один пример:

```bash
php examples/03-call-api.php
```

Запустите все примеры:

```bash
for file in examples/[0-9][0-9]-*.php; do php "$file"; done
```

Каждый пример использует фиксированные входные данные.
Каждый пример проверяет свой результат.
При ошибке проверки пример завершает работу с ненулевым статусом.

Не добавляйте пример документации, которому нужен реальный бот.
Используйте `RecordingClient` для примеров API.
Добавьте пример в `ExamplesTest`.

## Использование тестовых фабрик

Сгенерированные фабрики используют `fakerphp/faker`.
Приложение не устанавливает зависимости разработки библиотеки.
Установите Faker в приложение, которое использует фабрики:

```bash
composer require --dev fakerphp/faker
```

Укажите все значения, которые проверяет тест.
Позвольте Faker создать только значения, которые тест не проверяет.

Запустите пример фабрики:

```bash
php examples/05-test-factories.php
```

Установите свою фабрику объектов перед созданием первой фикстуры:

```php
Phenogram\Bindings\Factories\AbstractFactory::setFactory($factory);
```

`AbstractFactory` запрещает повторное назначение фабрики.
Устанавливайте фабрику один раз для каждого процесса PHP.

## Запуск рабочих тестов Telegram

Рабочие тесты запускаются только явно.
Они могут отправить документ в реальный чат.

Установите обязательные значения в окружении:

```bash
RUN_TELEGRAM_INTEGRATION=1 \
TELEGRAM_BOT_TOKEN='<token>' \
TEST_CHAT_ID='<chat-id>' \
vendor/bin/phpunit tests/Readme/ReadmeClientTest.php
```

Используйте отдельного тестового бота.
Используйте отдельный тестовый чат.
Не используйте рабочий токен.
Немедленно отзовите случайно раскрытый токен.

Стандартный набор пропускает эти тесты.
Флаг явного запуска предотвращает случайные сетевые запросы из локального файла `.env`.

## Запуск статического анализа

Запустите PHPStan:

```bash
composer phpstan
```

Текущая конфигурация проверяет `src` на уровне 5.
Она исключает файлы сериализатора.
Считайте это исключение техническим долгом.

## Проверка стиля кода

Запустите проверку без изменений:

```bash
tools/php-cs-fixer/vendor/bin/php-cs-fixer fix \
--dry-run \
--diff \
--sequential \
--config=.php-cs-fixer.php \
--allow-risky=yes
```

Запустите настроенный исправитель:

```bash
composer fix
```

Проверьте diff после автоматического исправления.
Сгенерированные файлы могут создать большой diff.

## Проверка изменения

Выполните следующие проверки перед отправкой изменения:

1. Запустите все исполняемые примеры.
2. Запустите PHPUnit.
3. Запустите PHPStan.
4. Запустите проверку стиля.
5. Запустите `composer lint:composer`.
6. Проверьте одинаковый объём документации на английском и русском языках.
